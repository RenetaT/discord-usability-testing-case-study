# Discord Usability Testing Case Study

A usability testing coursework project evaluating selected Discord features, including voice communication settings, communication hierarchy, and server management features.

## Academic Outcome

Mark: **71% – First Class**

Academic feedback highlighted appropriate usability goals, strong discussion of metrics, useful identification of possible errors, good procedure design, significant evidence of data collection, good results discussion, and excellent discussion of threats to validity.

## Project Overview

This study evaluated whether experienced Discord users could complete selected tasks across three areas:

- understanding and configuring voice communication settings
- navigating Discord’s server/channel/direct message hierarchy
- finding server management and server-specific identity settings

The study used task-based usability testing with direct observation, screen recording, and the think-aloud protocol.

## Usability Goals

### 1. Understandability of Voice Communication Settings

This goal examined whether users could understand Discord’s voice settings, particularly the difference between push-to-talk and voice activity, to avoid unintentionally broadcasting background noise.

### 2. Efficiency and Findability of Communication Hierarchy

This goal examined whether users could complete basic messaging tasks and understand the difference between servers, channels, direct messages, mentions, and search.

### 3. Findability of Server Management Features

This goal examined whether users could create a new server and change their display name only inside a specific server without confusing server-level and account-level settings.

## Methodology

The study used:

- direct observation
- screen recording
- think-aloud protocol
- task completion metrics
- time-on-task measurement
- mistake counting
- assistance/give-up tracking
- qualitative coding of observed behaviours and participant comments

## Participant Profile

The study focused on experienced Discord users rather than complete beginners. Participants were selected because they had prior experience using Discord regularly, joining servers, and using the platform for communication.

## Tasks Tested

### Task 1: Voice Communication Settings

Participants were asked to configure Discord so they would not unintentionally broadcast background noise during a voice call.

### Task 2: Communication Hierarchy and Findability

Participants completed messaging-related tasks, including:

- finding a server
- sending a message in the correct channel
- finding a previous Zoom link
- sending a direct message
- mentioning another user
- distinguishing between a server and a channel with a similar name

### Task 3: Server Management and Identity

Participants were asked to:

- create a new server named "Work Project"
- change their display name only inside that specific server
- check whether their original username remained unchanged elsewhere

## Key Findings

- Participants performed well on familiar communication tasks such as sending direct messages and using basic messaging features.
- Voice settings caused confusion because participants often treated the task as a microphone or volume issue rather than a transmission-mode issue.
- The server/channel distinction caused errors when similar names were used.
- Creating a server was easy, but changing a server-specific display name was harder to find.
- Some users confused global identity settings, server settings, and per-server profile settings.

## Main Usability Issues Identified

- unclear feedback around push-to-talk and voice activity
- confusion between microphone control and voice transmission mode
- weak distinction between servers and channels when names overlap
- unclear separation between global profile settings and server-specific identity settings
- users sometimes believed they had completed a task even when the action happened in the wrong location

## Recommendations

- Add clearer feedback when push-to-talk is active, such as “Push-to-talk is active: hold [key] to speak.”
- Make the current server/channel location more visually explicit.
- Improve labels around server-specific profile settings.
- Separate server settings and personal identity settings more clearly.
- Add clearer confirmation when a name change applies only to one server.

## Limitations

- The sample size was small.
- Participants used a test account rather than their own Discord accounts.
- A think-aloud protocol may have affected task completion time.
- Screen recording and observation may have influenced participant behaviour.
- The study focused only on experienced Discord users, so the findings may not represent first-time users.
